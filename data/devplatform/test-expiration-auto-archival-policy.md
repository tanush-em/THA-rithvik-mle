# Test Expiration and Auto-Archival Policy

## Overview

All DevPlatform assessments have a default expiration period of **30 calendar days** from the date of creation. After this period, the test is automatically archived and candidates can no longer access it.

## Key Points

- Tests expire 30 days after creation unless manually extended
- Expired tests are moved to the "Archived" section
- Candidates who haven't started the test will receive an "Assessment Expired" message
- Once a test expires, all pending invitations are automatically cancelled
- To extend a test, go to Settings > General > Expiration and set a new end date

## Reactivating Expired Tests

To reactivate an expired test:

1. Navigate to the Archived Tests section
2. Select the test you want to reactivate
3. Click "Reactivate" and set new expiration dates
4. Note: Previously cancelled invitations must be re-sent manually

## Important Notes

- Enterprise plan customers can set custom default expiration periods (7, 14, 30, 60, or 90 days)
- Tests with active candidates (those who have started but not submitted) will send a reminder 24 hours before expiration
- The archival process runs daily at midnight UTC
